Decision rules

Implemented analysis routes

Each method exposes the margin, uncertainty model, and acceptance criterion used by the backend.

Fixed-margin 95-95 Primary

Margin: d_NI = M1_lower * (1 - p)

Decision: non-inferior when the CI upper bound is <= d_NI.

Synthesis / putative-placebo Historical

Margin: inverse-variance pooled historical effects, then lower-bound margin.

Decision: non-inferior when the CI upper bound is <= d_NI,synthesis.

Bayesian NI Posterior

Model: Normal prior plus Normal likelihood for the worsening effect.

Decision: pass when P(effect <= d_NI) meets the configured threshold.

Equivalence Two-sided

Margin: symmetric +/- d around zero worsening effect.

Decision: equivalent when the full CI sits inside [-d, +d].

Mean-based Legacy

Margin: d_NI = M1 * (1 - p)

Decision: non-inferior when the point effect is <= d_NI.

Orientation

Direction convention

  • Lower-better metrics such as latency, energy, cost, and response time use treatment - baseline.
  • Higher-better metrics such as throughput and accuracy use baseline - treatment.
  • A larger worsening effect always means worse treatment performance.

Portfolio layer

Multi-objective decision rules

  • Strict: every selected metric must pass.
  • Weighted: normalized weights create an acceptability score.
  • Gatekeeper: mandatory metrics pass first, then weighted acceptability is applied.
